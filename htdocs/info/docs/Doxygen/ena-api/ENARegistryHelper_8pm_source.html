00553 
00554 =head1 SYNOPSIS
00555 
00556 # creation from a database server
00557 Bio::EnsEMBL::ENA::ENARegistryHelper->register_ena_dbs( $conf->{host},
00558        $conf->{port}, $conf->{user}, $conf->{pass}, $conf->{db});
00559 my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new();
00560 my $dbas = $helper->registry()->get_all_DBAdaptors();
00561 $dbas = $helper->get_by_taxon_id(388919);
00562 $dbas = $helper->get_by_name_pattern("Escherichia.*");

00564 # creation from a URL
00565 my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new(-URL=>'http://www.ebi.ac.uk/ena/ena_genomes_registry.json');
00566 
00567 =head1 DESCRIPTION
00568 
00569 This module is a helper that provides additional methods to aid navigating a registry of >2000 species across >10 databases. 
00570 It does not replace the Registry but provides some additional methods for finding species e.g. by searching for species that 
00571 have an alias that match a regular expression, or species which are derived from a specific ENA/INSDC accession, or species
00572 that belong to a particular part of the bacterial taxonomy. 
00573 
00574 There are a number of ways of creating a helper. The simplest and fastest is to supply a URL for a JSON resource that contains all the details
00575 needed to connect to the public ENA databases e.g.
00576 
00577     my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new(-URL=>'http://www.ebi.ac.uk/ena/ena_genomes_registry.json');
00578 
00579 Alternatively, a local file containing the required JSON can be specified instead:
00580 
00581     my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new(-FILE=>"/path/to/reg.json");
00582 
00583 Finally, a Registry already loaded with the ENA core databases can be supplied:
00584 
00585     my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new(-REGISTRY=>'Bio::EnsEMBL::Regsitry');
00586 
00587 If the standard Registry is used, the argument can be omitted completely:
00588 
00589     my $helper = Bio::EnsEMBL::ENA::ENARegistryHelper->new(-REGISTRY=>'Bio::EnsEMBL::Regsitry');
00590 
00591 To populate the registry with just the ENA databases for the current software release on a specified server, the following method can be used:
00592 
00593     Bio::EnsEMBL::ENA::ENARegistryHelper->register_ena_dbs( $host,
00594        $port, $user, $pass);
00595 
00596 Once a helper has been created, there are various methods to retreive DBAdaptors for species of interest:
00597 
00598 1. To find species by name - all DBAdaptors for species with a name or alias matching the supplied string:
00599 
00600     $dbas = $helper->get_by_name_exact('Escherichia coli str. K-12 substr. MG1655');
00601 
00602 2. To find species by name pattern - all DBAdaptors for species with a name or alias matching the supplied regexp:
00603 
00604     $dbas = $helper->get_by_name_exact('Escherichia coli .*);
00605 
00606 3. To find species with the supplied taxonomy ID:
00607 
00608     $dbas = $helper->get_by_taxon_id(388919);
00609     
00610 4. In coordination with a taxonomy node adaptor to get DBAs for all descendants of a node:
00611 
00612     my $node = $node_adaptor->fetch_by_taxon_id(511145);
00613     for my $child (@{$node_adaptor->fetch_descendants($node)}) {
00614         my $dbas = $helper->get_by_taxon_id($node->taxon_id())
00615         if(defined $dbas) {
00616             for my $dba (@{$dbas}) {
00617                 # do something with the $dba
00618             }
00619         }
00620     }
00621 
00622 The retrieved DBAdaptors can then be used as normal e.g.
00623 
00624     for my $gene (@{$dba->get_GeneAdaptor()->fetch_all_by_biotype('protein_coding')}) {
00625         print $gene->external_name."\n";
00626     }
00627 
00628 Once retrieved, the arguments needed for constructing a DBAdaptor directly can be dumped for later use e.g.
00629 
00630     my $args = $helper->dba_to_args($dba);
00631     ... store and retrieve $args for use in another script ... 
00632     my $resurrected_dba = Bio::EnsEMBL::DBSQL::DBAdaptor->new(@$args);
